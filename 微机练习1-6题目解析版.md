## 练习 1

### 1. 单选题（1.5 分）

51 单片机的机器周期由 12 个时钟周期组成，如果采用 8MHz 晶振作为时钟源，则一条 4 机器周期指令的执行时间为：

A. 2us  
B. 3us  
C. 6us  
D. 8us

**正确答案：C**

**解析：**

51 单片机 1 个机器周期 = 12 个时钟周期。8MHz 晶振的时钟周期为 `1 / 8MHz = 0.125us`，所以 1 个机器周期为 `12 × 0.125us = 1.5us`。题目是 4 机器周期指令，执行时间为 `4 × 1.5us = 6us`，选 C。

### 2. 填空题（3 分）

根据温度传感器 AT30TSE752A 的数据输出规则，回答下面问题：

1. 传感器输出值是 `0xB530` 时，它表示的温度值是：____
2. 传感器测得的温度值是 `-32.1875℃` 时，其输出值的十六进制表示为：____

注意：填空中按需要写正负号（`+/-`）、数字和小数点，不要写表示进制类型的前后缀或下标！

**正确答案：**

- 填空 1：`-74.8125`
- 填空 2：`DFD0`

**解析：**

AT30TSE752A 的温度输出按二进制补码表示，题目给出的数据低 4 位不是温度整数位，而是小数分辨率部分；每一格代表 `0.0625℃`，也就是 `1/16℃`。判断正负先看最高位：最高位为 1 表示负温度，需要按补码求幅值。

第 1 空：`0xB530` 最高位为 1，是负数。先求它对应的正数幅值：`0x10000 - 0xB530 = 0x4AD0`。因为最低 4 位表示 1/16℃，所以实际幅值为 `0x4AD0 / 16 = 0x04AD = 1197` 个 `0.0625℃`，即 `1197 × 0.0625 = 74.8125℃`。原数为负，所以结果是 `-74.8125`。

第 2 空：`-32.1875℃` 先求正数编码。`32.1875 / 0.0625 = 515`，十六进制为 `0x0203`；温度编码左移 4 位，得到正数形式 `0x2030`。负数用 16 位补码表示，所以 `0x10000 - 0x2030 = 0xDFD0`，填 `DFD0`。

### 3. 填空题（5 分）

某 64 位计算机存储系统中有符号整数数据类型规定为：`char` 为 8 位、`short` 为 16 位、`int` 为 32 位、`long` 为 64 位，存储模式为小端模式。下面数据块的各个成员按顺序从零地址单元开始，按照边界对齐原则依次向高地址单元存储，请根据数据块在存储器中的映像，完成下面的填空。

数据块：

```c
int m1 = 285;
long m2 = -1020;
char m3 = -10;
int m4 = -129;
```

注意：若单元未被使用或内容不确定，填 `XX`；用十六进制表示，填空中只需写数字，不要加表示进制的前后缀或下标！

内存映像填空位置：

| 地址 | +0 | +1 | +2 | +3 | +4 | +5 | +6 | +7 |
|---|---|---|---|---|---|---|---|---|
| `$0000` | (1) |  | (2) |  |  | (3) |  |  |
| `$0008` |  | (4) | (5) |  | (6) |  |  |  |
| `$0010` | (7) |  |  | (8) | (9) |  | (10) |  |

**正确答案：**

- 填空 1：`1D`
- 填空 2：`00` / `0`
- 填空 3：`XX` / `X`
- 填空 4：`FC`
- 填空 5：`FF`
- 填空 6：`FF`
- 填空 7：`F6`
- 填空 8：`XX` / `X`
- 填空 9：`7F`
- 填空 10：`FF`

**解析：**

这题要同时按“数据类型长度、边界对齐、小端模式、负数补码”四条规则排内存。已知 `char` 1 字节、`short` 2 字节、`int` 4 字节、`long` 8 字节；边界对齐通常要求变量起始地址是自身长度的整数倍；小端模式表示低有效字节放在低地址。

先放 `int m1 = 285`。`285 = 0x0000011D`，按小端排列为 `1D 01 00 00`，占 `$0000-$0003`，所以 `(1)=1D`，`(2)=00`。

接着放 `long m2`。`long` 要从 8 的整数倍地址开始，所以下一个可用地址不能是 `$0004`，要补齐到 `$0008`；`$0004-$0007` 是对齐填充区，内容不确定，所以其中对应的 `(3)=XX`。`m2=-1020`，先算正数 `1020=0x03FC`，64 位补码为 `0xFFFFFFFFFFFFFC04`，小端存放为 `04 FC FF FF FF FF FF FF`，地址 `$0008-$000F`，所以 `(4)=FC`，`(5)=FF`，`(6)=FF`。

再放 `char m3=-10`。8 位补码为 `0xF6`，放在 `$0010`，所以 `(7)=F6`。最后放 `int m4=-129`，`int` 需要 4 字节对齐，所以下一个起始地址应为 `$0014`，中间 `$0011-$0013` 是填充区，题中对应 `(8)=XX`。`-129` 的 32 位补码为 `0xFFFFFF7F`，小端存放为 `7F FF FF FF`，所以 `(9)=7F`，`(10)=FF`。

### 4. 填空题（4 分）

某二进制浮点数字长 20 位，其中阶码和尾数均用补码表示，阶码 7 位、尾数 13 位（均包含符号位），分别写出其所能表示的最大负数、最小规格化正数和最小非规格化正数。

（用十六进制表示，填空中只需写数字，不要加表示进制的前后缀或下标。）

1. 最大负数：____
2. 最小规格化正数：____
3. 最小非规格化正数：____

**正确答案：**

- 填空 1：`81FFF`
- 填空 2：`80800`
- 填空 3：`80001`

**解析：**

20 位浮点数由 7 位阶码和 13 位尾数组成，题目说明二者都用补码。7 位阶码补码范围是 `-64` 到 `+63`；13 位尾数补码中，正尾数形如 `0.xxxx`，负尾数形如 `1.xxxx`。最终十六进制答案就是把“7 位阶码 + 13 位尾数”拼成 20 位后写出。

第 1 空“最大负数”指最接近 0 的负数。要让负数绝对值尽量小，阶码取最小 `-64`，7 位补码为 `1000000`；尾数取负数中最接近 0 的值，13 位补码为 `1 111111111111`。拼接后得到二进制 `1000000 1111111111111`，按 4 位分组写成十六进制就是 `81FFF`。

第 2 空“最小规格化正数”要求正数且规格化。正数最小要取最小阶码 `-64`，阶码仍是 `1000000`；规格化正尾数最小为 `0.1000...0`，13 位尾数字段为 `0100000000000`。拼接后为 `1000000 0100000000000`，写成十六进制是 `80800`。

第 3 空“最小非规格化正数”允许尾数不满足规格化，只要是最小非零正尾数即可，所以阶码仍取 `-64`，尾数取最低位为 1 的 `0000000000001`。拼接得到 `1000000 0000000000001`，十六进制为 `80001`。

### 5. 填空题（2 分）

要使用 `ELPM` 指令访问 Flash 中地址为 `$A203` 单元的高字节，请问 `(Z) ← $____`，`(RAMPZ) ← $____`。

**正确答案：**

- 填空 1：`4407`
- 填空 2：`01`

**解析：**

AVR 的程序存储器通常按“字”组织，而 `LPM/ELPM` 指令实际用 `Z` 指针给出“字节地址”。一个程序存储器字有低字节和高字节，所以从字地址换成字节地址时要左移 1 位；访问高字节时最低位再加 1。

题目给出的 Flash 单元地址是 `$A203`，要访问这个单元的高字节：

`字节地址 = ($A203 << 1) + 1 = $14406 + 1 = $14407`

`ELPM` 使用 `RAMPZ:Z` 共同组成扩展程序存储器地址。`Z` 保存低 16 位，即 `$4407`；超出 16 位的高位 `$01` 放入 `RAMPZ`。因此 `(Z)=$4407`，`(RAMPZ)=$01`。

### 6. 单选题（1 分）
LPM指令的功能是以Z为指针从Flash存储器中读取一个字节数据，下面指令段中 `LPM` 指令读取的数据在 FLASH 中的地址为：

A. `0x01A6`

### 7. 填空题（2 分）

AVR 单片机中，内存堆栈输出如下图所示，若 `SP = $30dd`，执行指令 `POP R0` 后：

| 地址 | 内容 |
|---|---|
| ...... |  |
| `$30e0` | `0x35` |
| `$30df` | `0x34` |
| `$30de` | `0x33` |
| `$30dd` | `0x32` |
| `$30dc` | `0x31` |
| ...... |  |

1. `(R0) = $____`
2. `(SP) = $____`

**正确答案：**

- 填空 1：`33`
- 填空 2：`30de`

**解析：**

AVR 的栈是向低地址增长的：`PUSH` 时先让 `SP` 减小再写入数据；`POP` 时则反过来，先让 `SP` 增加，再从新的栈顶地址读出数据。这里初始 `SP=$30dd`，执行 `POP R0` 的关键步骤是：

1. `SP ← SP + 1`，所以 `$30dd + 1 = $30de`。
2. 从新地址 `$30de` 读出一个字节送入 `R0`。
3. 查表可知 `$30de` 单元内容为 `0x33`。

因此 `R0` 得到 `$33`，执行后栈指针停在 `$30de`。注意不是读取 `$30dd` 的 `0x32`，因为 `POP` 对 AVR 来说是先调整 SP，再读数据。

### 8. 填空题（8 分）

已知：`(R0)=0x23`，`(R1)=0x22`，`(R2)=0x23`，`(R3)=0x24`，分析下面汇编指令段：

```asm
1. ldi R27, 0
2. ldi R26, $F3
3. st X+, R0
4. st X, R1
5. ldi R26, $F6
6. st X, R2
7. st -X, R3
```

第 3 行中，源操作数是（1）____，寻址方式：（2）____；目的操作数是（3）____，寻址方式：（4）____。  
第 5 行中，源操作数是（5）____，寻址方式：（6）____。  
第 7 行指令中，`X = 0x`（7）____。  
第 7 行指令执行后，DATA RAM 内容映像为：

| 地址 | 内容 |
|---|---|
| `$00F3` |  |
| `$00F4` | `0x`（8） |
| `$00F5` | `0x`（9） |
| `$00F6` | `0x`（10） |

**正确答案：**

- 填空 1：`R0`
- 填空 2：`寄存器寻址`
- 填空 3：`X`
- 填空 4：`寄存器间接寻址`
- 填空 5：`F6`
- 填空 6：`立即数`
- 填空 7：`00F5`
- 填空 8：`22`
- 填空 9：`24`
- 填空 10：`23`

**解析：**

这题要跟踪 X 指针和每条 `st` 指令的副作用。AVR 中 X 指针由 `R27:R26` 组成，即 `XH=R27`，`XL=R26`。

第 1、2 行执行后：`R27=0x00`，`R26=0xF3`，所以 `X=$00F3`。第 3 行 `st X+, R0` 的源操作数是 `R0`，属于寄存器寻址；目的操作数是 `X` 指向的 SRAM 单元，属于寄存器间接寻址。它先把 `R0=0x23` 写入 `$00F3`，再让 X 后增为 `$00F4`。所以第 3 行的源、目的及寻址方式分别对应填空 1-4。

第 5 行 `ldi R26, $F6` 是立即数送寄存器，源操作数是立即数 `F6`，寻址方式是立即数寻址。此时 `R27` 没变仍为 `0x00`，所以 X 被改成 `$00F6`。第 6 行 `st X, R2` 把 `R2=0x23` 写入 `$00F6`，X 不变。

第 7 行 `st -X, R3` 是预减间接寻址：执行存储前先 `X ← X - 1`，所以 `$00F6` 先变为 `$00F5`，再把 `R3=0x24` 写入 `$00F5`。因此第 7 行中实际用于存储的 `X=$00F5`。最终相关 DATA RAM 为：`$00F3=23`，`$00F4=22`，`$00F5=24`，`$00F6=23`；题表只要求 `$00F4-$00F6`，所以填 `22、24、23`。

## 练习 5

### 1. 填空题（20 分）

已知 DATA RAM 内存中地址 `$8020` 开始的若干内存单元中的数据依次为：`13, 4, 61, 87, 255, 124, 37, 132`；代码运行前，所有寄存器初值均为 0，试分析如下汇编指令代码的执行过程：

```asm
1  L0: ldi r17, 8
2      ldi XL, $20
3      ldi XH, $80
4  L1: ld  r20, X+
5      ldi r18, 7
6  L2: lsr r20
7      brcs L3
8      inc r19
9  L3: dec r18
10     brne L2
11     dec r17
12     brne L1
13     ret
```

填空：

1. 第 4 行第一次执行后，`(X) = $____`；
2. 第 4 行第一次执行后，`(r20) = $____`；
3. 第 6 行第一次执行后，`(r20) = $____`；
4. 第 11 行第一次执行后，`(r19) = $____`；
5. 执行到第 13 行时，`(r19) = $____`；
6. 执行到第 13 行时，`(r20) = $____`；
7. 执行到第 13 行时，`(r26) = $____`；
8. 执行到第 13 行时，`(r27) = $____`；
9. 全部代码执行完成后，`L1` 的循环次数 = ____；
10. 全部代码执行完成后，`L2` 的循环次数 = ____。

**正确答案：**

- 填空 1：`8021`
- 填空 2：`0D` / `d` / `13`
- 填空 3：`06` / `6`
- 填空 4：`04` / `4`
- 填空 5：`1A` / `26`
- 填空 6：`01` / `1`
- 填空 7：`28`
- 填空 8：`80`
- 填空 9：`8` / `08`
- 填空 10：`56` / `38`

**解析：**

第 1 行设置外层循环次数 `r17=8`，表示要处理 8 个内存数据；第 2、3 行设置 X 指针，`XH:XL=$80:$20`，所以 `X=$8020`。题目说内存从 `$8020` 开始依次为十进制 `13,4,61,87,255,124,37,132`。

第一次执行第 4 行 `ld r20, X+` 时，从 `$8020` 读出十进制 `13`，即十六进制 `$0D`，送入 `r20`；同时 X 后增到 `$8021`，所以填空 1 为 `8021`，填空 2 为 `0D`。第 6 行第一次 `lsr r20` 是逻辑右移，`0000 1101` 右移后变为 `0000 0110`，即 `$06`，原最低位 `1` 进入 C 标志，所以填空 3 为 `06`。

内层循环 L2 每处理一个数据执行 7 次。`lsr` 每次把当前最低位移入 C；第 7 行 `brcs L3` 表示如果 C=1 就跳过 `inc r19`，如果 C=0 才执行 `inc r19`。因此 `r19` 统计的是这 8 个数低 7 位中“0”的个数。第一个数 `13` 的低 7 位是 `0001101`，其中有 4 个 0，所以第一次外层循环结束、执行到第 11 行时 `r19=04`，填空 4 为 `04`。

全部 8 个数都处理后，逐个统计低 7 位中 0 的个数：`13` 有 4 个，`4` 有 6 个，`61` 有 2 个，`87` 有 3 个，`255` 有 0 个，`124` 有 2 个，`37` 有 4 个，`132` 的低 7 位为 `0000100` 有 5 个，合计 `4+6+2+3+0+2+4+5=26`，十六进制为 `1A`，所以填空 5 为 `1A`。每个数右移 7 次后只剩原最高位进入最低位；最后一个数 `132=1000 0100b`，右移 7 次后为 `0000 0001`，所以填空 6 为 `01`。

X 指针每次外层循环在第 4 行后增 1，共处理 8 个数据，所以从 `$8020` 变为 `$8028`。X 由 `r27:r26` 组成，因此 `r26=$28`，`r27=$80`，对应填空 7、8。外层 L1 循环 8 次；内层 L2 每个数据 7 次，总次数 `8×7=56`，十六进制为 `38`，所以最后两个填空可写 `8` 和 `56` 或 `38`。

## 练习 6

### 1. 填空题（20 分）

分析下面汇编程序代码，填空回答问题。

```asm
1  su1: ldi r31, high(t1<<1)
2       ldi r30, low(t1<<1)
3       ldi r29, $03
4       mov r28, r30
5  d0:  lpm r20, Z+
6       andi r20, $FE
7       st Y+, r20
8       subi r30, -1
9       cpi r30, $00
10      brne d0
11      ret
12 .org $0941
13 t1:  .dw $0011, $2233, $4455, $6677
14      .dw $8899, $aabb, $ccdd, $eeff
```

填空：

1. 第 4 行执行后，`(Z) = $____`；
2. 第 4 行执行后，`(Y) = $____`；
3. 第 5 行第一次执行后，`(r20) = $____`；
4. 第 5 行第一次执行后，`(Z) = $____`；
5. 第 7 行第一次执行后，`(Y) = $____`；
6. 第 8 行第一次执行后，`(r30) = $____`；

第 11 行执行后，DATA memory 映像为：

| 地址 | 内容 |
|---|---|
| `$0385` | `$`（7） |
| `$0386` | `$`（8） |
| `$0387` | `$`（9） |
| `$0388` | `$`（10） |

**正确答案：**

- 填空 1：`1282`
- 填空 2：`0382`
- 填空 3：`11`
- 填空 4：`1283`
- 填空 5：`0383`
- 填空 6：`84`
- 填空 7：`76`
- 填空 8：`98`
- 填空 9：`BA`
- 填空 10：`DC`

**解析：**

`.org $0941` 表示表 `t1` 位于程序存储器字地址 `$0941`。AVR 的 `LPM` 用 Z 指针访问程序存储器的字节地址，所以代码中使用 `t1<<1`。因此 `t1<<1 = $0941 × 2 = $1282`，第 1、2 行后 `Z=$1282`，填空 1 为 `1282`。第 3 行令 `r29=$03`，第 4 行 `mov r28,r30`，此时 `r30=$82`，所以 `Y=r29:r28=$0382`，填空 2 为 `0382`。

第 5 行第一次 `lpm r20, Z+` 从程序存储器字节地址 `$1282` 读出数据。表 `t1` 的第一个字是 `.dw $0011`，按低字节在前，`$1282` 处是 `$11`，所以填空 3 为 `11`。由于使用 `Z+`，读完后 Z 自动加 1，变为 `$1283`，填空 4 为 `1283`。

第 6 行 `andi r20,$FE` 会把最低位清 0，第一次 `11 & FE = 10`。第 7 行 `st Y+, r20` 把 `$10` 写入 `$0382`，然后 Y 后增为 `$0383`，所以填空 5 为 `0383`。第 8 行 `subi r30,-1` 等价于 `r30 = r30 + 1`，此时 `r30` 从 `$83` 变为 `$84`，所以填空 6 为 `84`。这一步很关键：一次循环中 Z 先因 `LPM Z+` 加 1，又因第 8 行低字节再加 1，所以实际每轮跳过一个字节。

表中 `.dw $0011,$2233,$4455,$6677,$8899,$aabb,$ccdd,$eeff` 在程序存储器中按字节依次可看作 `11,00,33,22,55,44,77,66,99,88,BB,AA,DD,CC,FF,EE`。由于程序每轮隔一个字节读取，所以依次读到低字节 `11,33,55,77,99,BB,DD,FF...`。再经过 `andi $FE` 清最低位后，写入数据变为 `10,32,54,76,98,BA,DC,FE...`，从 `$0382` 开始连续写入。于是 `$0385` 是第 4 个写入值 `76`，`$0386=98`，`$0387=BA`，`$0388=DC`。
